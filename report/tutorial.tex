\section{Getting Started}

The system runs on PHP and MySQL. There are two ways to set it up depending on what you have installed.

\subsection{Option 1: Using the startup script (recommended)}

If you have WSL (Windows Subsystem for Linux) or a Linux machine, this is the easiest way. Open a terminal, go to the project folder, and run:

\begin{lstlisting}[style=code]
./startscript.sh
\end{lstlisting}

The script will install PHP and MySQL if they are not already installed, set up the database with sample data, and start a local web server. Once you see ``Ready! Open http://localhost:8000'' you can open that URL in your browser.

To stop the server, press Ctrl+C in the terminal.

To run the automated tests (checks that all pages load, forms submit, and queries return data):

\begin{lstlisting}[style=code]
./test.sh
\end{lstlisting}

\subsection{Option 2: Using WAMP}

If you prefer WAMP on Windows:

\begin{enumerate}
  \item Install WAMP Server from \texttt{wampserver.com}
  \item Copy the project folder to \texttt{C:\textbackslash wamp64\textbackslash www\textbackslash JonesAuto\textbackslash}
  \item Start WAMP and wait for the tray icon to turn green
  \item Open phpMyAdmin at \texttt{http://localhost/phpmyadmin/} (login: root, no password)
  \item Create a database called \texttt{jonesauto}
  \item Import \texttt{db\_setup.sql} first (creates the tables), then \texttt{db\_seed.sql} (loads sample data)
  \item Open \texttt{http://localhost/JonesAuto/} in your browser
\end{enumerate}

\section{Home Page}

When you open the system you will see the home page with three stats at the top showing how many cars are available, how many customers are in the system, and how many sales have been made. Below that are links to all the forms on the left and all the reports on the right. There is also a link to the queries page at the bottom.

The navigation bar at the top of every page has dropdown menus for Forms and Reports so you can get to any page from anywhere.

\section{Using the Forms}

There are 6 input forms. Here is what each one does.

\textbf{Purchase Form} -> Records a car bought at auction. Pick a buyer from the dropdown, fill in the purchase info and vehicle details, and add up to 5 repair problems at the bottom. Click ``Record Purchase'' to save.

\textbf{Sale Form} -> Records selling a car to a customer. You can pick an existing customer from the dropdown or enter a new one. Select an available vehicle, fill in the sale price and financing details, and add employment history for the customer. The car gets marked as sold automatically.

\textbf{Warranty Form} -> Adds warranty coverage to a sale. Pick a sale from the dropdown and the vehicle/customer info fills in automatically. Add up to 3 warranty items with type, dates, cost, and what is covered.

\textbf{Payment Form} -> Records a customer payment. Pick a customer first and the page shows their info and existing payments. Then fill in the payment details. If the payment is late the system updates the customer's late payment stats automatically.

\textbf{Customer Form} -> Adds a new customer. Fill in their info and click submit. All existing customers are shown in a table below the form.

\textbf{Employee Form} -> Adds a new employee (buyer or salesperson). Same layout as the customer form.

\section{Viewing Reports}

There are 6 reports. Most have a filter at the top.

\begin{itemize}
  \item \textbf{Current Inventory} -> Cars available on the lot. You can filter by make.
  \item \textbf{Sales Summary} -> All sales with profit calculations. Filter by date range.
  \item \textbf{Payment History} -> Pick a customer to see all their payments. Late ones show up in red.
  \item \textbf{Repair Costs} -> Estimated vs actual repair costs. Over-budget repairs are highlighted.
  \item \textbf{Active Warranties} -> All warranty items with expiry dates. Red = expired, yellow = expiring soon.
  \item \textbf{Late Payments} -> Customers with late payments, sorted worst first.
\end{itemize}

\section{Running Queries}

Go to the queries page from the nav bar. There is a dropdown with 12 business queries. Pick one and click ``Run Query'' to see the results. The SQL is shown on the page so you can see what is running. Query 3 has an extra text box where you type a car make to search for.

\section{Troubleshooting}

\begin{itemize}
  \item If you get a database connection error, make sure MySQL is running
  \item If using the startup script, make sure you have sudo access
  \item If using WAMP, import \texttt{db\_setup.sql} before \texttt{db\_seed.sql} (order matters)
  \item If pages are blank or show PHP errors, check that \texttt{config.php} has the right database credentials
\end{itemize}
